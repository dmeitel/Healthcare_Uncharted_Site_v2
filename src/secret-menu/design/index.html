Design Reference

The parts catalog

Every shared object the tools are built from, rendered dead on the table with its class name pinned to it. Nothing on this page functions. The words behind it live in docs/HU-UI-GRAMMAR.md; the code lives in hu-global.css and hu-kit.js. If a tool ships a surface that is not on this page, it either joins the catalog or it does not ship.

Specimens render from the REAL shared classes, so this page and the kit cannot quietly disagree.

05The ten budget rules
The teeth. Classification without budgets is decoration.
Phone screen budget: canvas + nav + toolbar + at most ONE transient surface. Opening one closes the others. Desktop gets two.
Canvas gestures declutter: pan or zoom means "show me the canvas." HUDs fade, cards drop to peek, guides collapse.
One bottom sheet site-wide, three detents, grabber plus X, nonmodal at peek and half.
One zoom contract per family. Geo: pinch + drag + wheel, locate FAB, fit. Boards: the career-tree contract.
Motion: 250ms ceiling on phones, desktop keeps its glide, reduced-motion always honored.
Touch floor: 44px targets, primary actions in the bottom half of the screen.
Weight: nothing over ~300KB before first paint. Data loads on intent with a visible signal.
GPS: locate FAB only, permission on tap, high accuracy off, manual fallback. Nothing leaves the browser.
Modals: one at a time, blocking only for onboarding or data-loss confirmation.
Guidance leaves: every hint collapses on demonstrated use, not on a timer.
